### Why an Antigravity App is "World-Class" vs. a Gemini Gem

1. **Hard Constraint Enforcement (The "Anti-Hallucination" Layer):** LLMs, including Gemini 3, are probabilisticâ€”they sometimes guess at character counts. In Antigravity, you can instruct the agent to write a Python script that *mathematically counts* the characters in a hook. If the hook is 211 characters, the script rejects it and forces a rewrite automatically. A Gem cannot do this.
2. **Production of Final Assets (PDF Generation):** A Gem can only give you the *text* for a carousel. An Antigravity app can be instructed to use a library like `ReportLab` or `Fpdf` to take that text and actually **render the downloadable PDF file** with your branding, colors, and typography.
3. **Active Trend Hunting (Live Research):** A Gem relies on its training data or standard Google Search integration. An Antigravity agent can be programmed to proactively "Newsjack" by scraping specific industry sites, summarizing the top 3 stories from the last 6 hours, and feeding that directly into the content generator.

### The Prompts to Build This App (For Google Antigravity)

To build this, you would open Google Antigravity and use its "Manager Surface" or chat interface to define the mission. You don't need to write the code; you prompt the agent to build the app for you.

Here is the step-by-step prompting strategy to build the **"LinkedIn Command Center."**

#### Phase 1: The Foundation (Build the UI and Logic)

*Paste this prompt into Antigravity to create the core application.*

> **Prompt:**
> "Build a Python web application using Streamlit called 'The LinkedIn Command Center.'
> **Core Functionality:**
> 1. Create a sidebar with a 'Mode Selector' containing three options: 'The Architect (Thought Leadership)', 'The Kineticist (Viral Growth)', and 'The Structurist (Carousels)'.
> 2. Main Input: A text area for 'Topic or Rough Draft'.
> 3. **The Logic:** When the user clicks 'Generate', send the input to the Gemini 3 Pro API. You must dynamically swap the System Instruction based on the selected mode.
> 4. **System Instructions:** I will provide the specific XML system prompts for each mode. Create a `prompts.py` file to store them as string constants.
> 5. **Quality Gate (Crucial):** Before displaying the output, write a Python function `verify_constraints(text)` that checks if the first two lines (the hook) are under 210 characters. If they are not, automatically re-prompt Gemini to shorten the hook without showing the user the failed attempt."
> 
> 

#### Phase 2: The "Trend Hunter" Engine (Adding Live Research)

*Once the base app is running, use this prompt to add the research capability.*

> **Prompt:**
> "Add a 'Trend Hunter' feature to the application.
> **Functionality:**
> 1. Add a checkbox in the UI labeled 'Newsjack Mode'.
> 2. When checked, before generating the post, use an autonomous browsing agent to search Google News for the input topic (limit to the last 24 hours).
> 3. Extract the top 3 headlines and their core arguments.
> 4. Inject these headlines into the User Prompt sent to Gemini with the instruction: 'Context: Here is the breaking news. Use the "Negative Contrarian" hook pattern to pivot off this news.'"
> 
> 

#### Phase 3: The "Visualizer" (Automated PDF Creation)

*This is the "world-class" differentiator. Use this prompt to make the app generate actual files.*

> **Prompt:**
> "Upgrade 'The Structurist' mode to generate a downloadable file.
> **Requirements:**
> 1. When 'The Structurist' is selected, instruct Gemini to output the content as structured JSON (Slide Number, Headline, Body Text).
> 2. Write a Python script using the `ReportLab` library or `Streamlit-PDF` that parses this JSON.
> 3. The script should generate a 1080x1350 (4:5 ratio) PDF.
> 4. Style the PDF with a black background, white text, and a large bold font for headlines.
> 5. Display a 'Download Carousel PDF' button in the app so I can get the final asset immediately."
